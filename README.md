# vmb-rs

Rust bindings for [Allied Vision](https://www.alliedvision.com/)'s **Vimba X**
machine-vision SDK (`VmbC` C API).

This workspace follows a ports-and-adapters layout: the domain is generic
over a `VmbRuntime` trait; the production FFI adapter and an in-memory
fake for unit tests each implement it.

| Crate | Description |
| --- | --- |
| [`vmb-sys`](./vmb-sys) | Raw `extern "C"` FFI bindings generated by `bindgen`. Thin linking shim. |
| [`vmb-core`](./vmb-core) | Runtime-agnostic domain types (`VmbError`, `Frame`, `PixelFormat`, `CameraInfo`, `DiscoveryEvent`) and the `VmbRuntime` port. **No `unsafe`, no `vmb-sys` dependency.** |
| [`vmb-ffi`](./vmb-ffi) | Production `VmbRuntime` implementation that links against `libVmbC`. The only crate that contains `unsafe` FFI calls. |
| [`vmb-fake`](./vmb-fake) | Programmable in-memory `VmbRuntime` for unit-testing downstream code. |
| [`vmb`](./vmb) | Facade crate — re-exports the domain + adapter and provides a `real()` convenience constructor. **Most users depend on this crate.** |

All crates are dual-licensed under **MIT OR Apache-2.0** and published to
crates.io.

## Installation

```toml
[dependencies]
vmb = { version = "0.2", features = ["sdk"] }
```

The `sdk` feature links against the Vimba X SDK at build time. Install the
SDK from Allied Vision first and set `VIMBA_X_HOME` to its install prefix
(Linux) or rely on the default `/Library/Frameworks/VmbC.framework` location
(macOS).

Without the `sdk` feature the crate compiles to a stub so downstream
workspaces can build on machines that do not have the SDK installed — useful
for CI, dev environments, and contributors without hardware access.

See each crate's README for details:
- [vmb-sys/README.md](./vmb-sys/README.md) — SDK discovery, regenerating bindings
- [vmb-core/README.md](./vmb-core/README.md) — domain types + `VmbRuntime` port
- [vmb-ffi/README.md](./vmb-ffi/README.md) — production FFI adapter
- [vmb-fake/README.md](./vmb-fake/README.md) — test adapter
- [vmb/README.md](./vmb/README.md) — safe API usage

## Usage

```rust,ignore
use vmb::{DiscoveryEvent, Frame};

let system = vmb::real()?;                         // starts the Vimba runtime
let cameras = system.list_cameras()?;
let mut camera = system.open_camera(&cameras[0].id)?;
camera.load_settings(std::path::Path::new("day.xml"))?;

let _reg = system.register_discovery(|ev: DiscoveryEvent| {
    println!("discovery: {ev:?}");
})?;

camera.start_capture(4, |frame: &Frame<'_>| {
    // copy bytes out before returning — the adapter re-queues the buffer.
    let _bytes = frame.to_vec();
})?;
```

Unit tests in downstream code can swap the production adapter for
`vmb_fake::FakeVmbRuntime` to exercise error paths and teardown
sequences without hardware.

## Platforms

Tested on:
- Linux x86_64 (Vimba X 2023-4 and later)
- Linux aarch64
- macOS (VmbC framework)

Windows support is possible but not currently exercised.

## Versioning

This workspace follows semantic versioning. While the major version is `0`:
- Breaking changes bump the MINOR digit (0.1.x → 0.2.0)
- Non-breaking additions bump PATCH (0.1.0 → 0.1.1)

Releases are tag-triggered (`v*.*.*` tags on `main`) and automatically
published to crates.io by GitHub Actions.
