# vmb-sys

Raw FFI bindings to Allied Vision's **Vimba X** C API (VmbC).

This crate exposes the C type definitions generated by
[`bindgen`](https://github.com/rust-lang/rust-bindgen) from the Vimba X SDK
headers plus a runtime-loaded [`VmbApi`] handle. All function pointers are
resolved at program startup via [`libloading`] â€” the crate has **no
build-time dependency** on the Vimba X SDK.

It is consumed by the safe `vmb` wrapper crate â€” most users should depend
on `vmb`, not on `vmb-sys` directly.

[`libloading`]: https://crates.io/crates/libloading
[`VmbApi`]: crate::VmbApi

## Runtime library discovery

`VmbApi::load()` tries these paths in order:

1. `$VIMBA_X_HOME/api/lib/<arch>/libVmbC.<ext>` (older multi-arch installers)
2. `$VIMBA_X_HOME/api/lib/libVmbC.<ext>` (newer single-arch installers)
3. `$VIMBA_X_HOME/VmbC.framework/VmbC` (macOS, custom framework location)
4. `/Library/Frameworks/VmbC.framework/VmbC` (macOS default)
5. The bare file name (`libVmbC.so` / `libVmbC.dylib` / `VmbC.dll`) â€” the
   platform's default dynamic linker search path (`LD_LIBRARY_PATH`,
   `/etc/ld.so.conf`, `DYLD_LIBRARY_PATH`).

If every candidate fails, `VmbApi::load()` returns
`VmbLoadError::LibraryOpen` naming the last path tried and the underlying
OS error.

### macOS

The Vimba X framework installs to `/Library/Frameworks/VmbC.framework`
by default. Override via `VIMBA_X_HOME` if you installed to a custom
location.

### Linux

Set `VIMBA_X_HOME` to the SDK install prefix:

```sh
export VIMBA_X_HOME=/opt/VimbaX_2023-4
```

Alternatively, add the SDK lib directory to `LD_LIBRARY_PATH` or install
a symlink into `/usr/local/lib` so the bare `libVmbC.so` fallback
succeeds.

## Regenerating bindings

The bindings in `src/bindings.rs` are **committed to the repository** and are
*not* regenerated at build time. This keeps builds deterministic, avoids
requiring `bindgen` (or the SDK headers) on every developer machine, and
makes the crate publishable to crates.io.

To refresh the bindings after an SDK upgrade, run `bindgen` against
`wrapper.h` with the Vimba X include path:

```sh
bindgen wrapper.h \
  -o src/bindings.rs \
  --allowlist-function 'Vmb.*' \
  --allowlist-type 'Vmb.*' \
  --allowlist-var 'Vmb.*' \
  -- -I"${VIMBA_X_HOME}/api/include"
```

After regeneration, manually strip the `unsafe extern "C" { ... }` block
at the end of `bindings.rs` â€” those declarations are replaced at runtime
by `VmbApi` fields in `src/api.rs`. Then update `api.rs` to reflect any
changed signatures. Review the diff carefully before committing.
