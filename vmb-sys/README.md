# vmb-sys

Raw FFI bindings to Allied Vision's **Vimba X** C API (VmbC).

This crate exposes `extern "C"` declarations and C types generated by
[`bindgen`](https://github.com/rust-lang/rust-bindgen) from the Vimba X SDK
headers. It is consumed by the safe `vmb` wrapper crate — most users should
depend on `vmb`, not on `vmb-sys` directly.

## Feature flags

| Feature | Default | Description |
| --- | --- | --- |
| `sdk`   | off     | Enables SDK discovery and linking against `libVmbC` / `VmbC.framework`. |

**Without `sdk`** the crate compiles to a stub rlib: the `extern "C"`
declarations are present but no link directives are emitted, so
`cargo build --workspace` succeeds on machines that do not have the Vimba X
SDK installed. This is intentional so CI, development environments, and
contributors without hardware access can still build the workspace.

**With `sdk`** the build script locates the SDK and emits
`cargo:rustc-link-lib` directives. A final binary that actually calls into
`vmb-sys` must be built with `--features vmb-sys/sdk` (typically via the
safe wrapper's own `sdk` feature).

## SDK discovery

### macOS

The build script expects the Vimba X framework at
`/Library/Frameworks/VmbC.framework`. Install the Vimba X SDK from Allied
Vision and it will land there by default.

### Linux

The build script looks for the SDK via the `VIMBA_X_HOME` environment
variable, falling back to `/usr/local/VimbaX_2023-4` if unset. Headers are
expected under `$VIMBA_X_HOME/api/include/VmbC/` and libraries under
`$VIMBA_X_HOME/api/lib/{x86_64,arm64,arm}/`.

```sh
export VIMBA_X_HOME=/opt/VimbaX_2023-4
cargo build -p vmb-sys --features sdk
```

## Regenerating bindings

The bindings in `src/bindings.rs` are **committed to the repository** and are
*not* regenerated at build time. This keeps builds deterministic and avoids
requiring `bindgen` (or the SDK headers) on every developer machine.

To refresh the bindings after an SDK upgrade:

```sh
scripts/regenerate-vmb-bindings.sh
```

The script wraps a `bindgen` invocation against `wrapper.h` and writes the
result back to `vendor/vmb-sys/src/bindings.rs`. Review the diff carefully
before committing.
